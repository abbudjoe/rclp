# PMF Research Guardrails

This repo can earn technical conversations, but it does not prove market demand.

## Questions to validate

- Do operators have an authority gap between fleet-level systems and robot-local execution?
- Do they gate remote assist based on network quality?
- Do they bind software actor identity to robot capabilities?
- Can they revoke one autonomy module/agent without disabling the robot?
- Can they reconstruct agent/operator/robot/network/policy state after an incident?
- Would they run an edge daemon or command gate?

## Strong signal

- Customer asks for a pilot on a real robot or simulator.
- Customer says incident reconstruction or remote-assist gating is painful today.
- Customer provides logs, network traces, or a robot-stack integration path.

## Weak signal

- “Interesting protocol.”
- “Maybe useful in the future.”
- “We already solve this internally.”
- “Can you also sell us cheaper data?”
